# Game Data Platform - Backend

ระบบ Backend สำหรับแพลตฟอร์มขายข้อมูลเกม พัฒนาด้วย Go + Gin และ PostgreSQL โดยเน้นความเรียบง่ายและประสิทธิภาพ (ไม่ใช้ ORM)

## โครงสร้างโปรเจกต์
- `app/`: ไฟล์ซอร์สโค้ดทั้งหมดของ Go
  - `main.go`: จุดเริ่มต้นของโปรแกรมและจุดลงทะเบียน Routes
  - `db/`: การเชื่อมต่อฐานข้อมูล
  - `user/`: โมดูลการจัดการผู้ใช้ (สมัครสมาชิก, เข้าสู่ระบบ, เติมเงิน)
  - `pkg/`: โมดูลแพ็กเกจและการสมัครสมาชิก (อัปเกรด, ต่ออายุ)
- `initdb/`: ไฟล์ SQL สำหรับสร้างฐานข้อมูลเริ่มต้น
- `docker-compose.yml`: การตั้งค่าฐานข้อมูล PostgreSQL

## การติดตั้งและเริ่มใช้งาน

### 1. เตรียมฐานข้อมูล (Docker)
ตรวจสอบให้แน่ใจว่าได้ติดตั้ง Docker แล้ว จากนั้นรันคำสั่ง:
```bash
docker compose up -d
```
*ฐานข้อมูลจะรันที่ Port **5435** (เพื่อเลี่ยงการชนกับ Postgres ตัวทั่ไป)*

### 2. ตั้งค่าสภาพแวดล้อม (.env)
ไฟล์ `.env` ในโฟลเดอร์ `app/` ควรมีค่าดังนี้:
```env
DB_HOST=127.0.0.1
DB_PORT=5435
DB_USER=admin
DB_PASSWORD=admin1234
DB_NAME=game_data_platform
PORT=8080
```

### 3. รันโปรเจกต์ (Go)
เข้าไปที่โฟลเดอร์ `app/` และรันด้วย `air` (เพื่อระบบ Hot Reload):
```bash
cd app
air
```
หรือรันปกติ:
```bash
go run main.go
```

## ระบบที่สำคัญ

### 1. การสมัครสมาชิกและกระเป๋าเงิน (User & Wallet)
- ผู้ใช้สามารถสมัครสมาชิกและเติมเงินเข้า Balance ได้
- Balance จะถูกใช้ในการซื้อหรืออัปเกรดแพ็กเกจ

### 2. ระบบ Upgrade & Renewal (การอัปเกรดและต่ออายุ)
- **Upgrade**: หากมีแพ็กเกจเดิมอยู่แล้วเปลี่ยนเป็นอันที่แพงกว่า จะจ่ายเฉพาะ **"ส่วนต่าง"** และเริ่มนับเวลาใหม่
- **Renewal**: หากซื้อแพ็กเกจเดิมซ้ำ จะจ่ายราคาเต็มและ **"บวกเวลาเพิ่ม"** จากของเดิม
- **Dev Mode**: ปรับวันใช้งานของทุกแพ็กเกจเป็น **1 วัน** เพื่อให้ทดสอบได้รวดเร็ว

## รายการ API (Endpoints)

| Method | Path | คำอธิบาย |
|---|---|---|
| POST | `/api/users/register` | สมัครสมาชิกใหม่ |
| POST | `/api/users/login` | เข้าสู่ระบบ |
| GET | `/api/users/:id` | ดูโปรไฟล์และยอดเงิน |
| POST | `/api/users/:id/topup` | เติมเงินเข้ากระเป๋า |
| GET | `/api/packages` | รายการแพ็กเกจทั้งหมด |
| POST | `/api/packages/purchase` | ซื้อ/อัปเกรดแพ็กเกจ (ผ่าน `user_id` query) |
| GET | `/api/packages/subscription` | ดูแพ็กเกจปัจจุบันที่ใช้งานอยู่ |

## การทดสอบ
สามารถนำไฟล์ `GameDataPlatform.postman_collection.json` ในโฟลเดอร์ `app/` ไป Import เข้าโปรแกรม Postman เพื่อทดสอบ API ทั้งหมดได้ทันที
